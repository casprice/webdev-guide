Responsive Design

The idea of have "responsive design" on a website means that the site is well-displayed across multiple platform, from widescreen monitors to mobile smart phones, using "media queries". Media queries should be at the bottom of the stylesheet so as to make more specific instructions for certain device dimensions.

Mobile Styles

For an individualized narrow, or "mobile-style" layout, add the following to the the CSS stylesheet.

@media only screen and (max-width: 400px) {
   /* Mobile-only styles go here */
 }

The @media part begins the conditional statement. The only screen "media type" means that the contained styles should only be applied to devices with screens, as opposed to printed documents. Finally, the specifications in parentheses defines the device dimensions you're targeting.

Typically, you should start designing based on the mobile layout and work your way up to the desktop version, because desktop layouts are more complex. Additionally, mobile devices tend to agree more with the default vertical organization of containers on a webpage.

Tablet Styles

For an individualized medium, or "tablet-style" layout, add the following to the CSS stylesheet.

@media only screen and (min-width: 401px) and (max-width: 960px) {
   /* Tablet-only styles go here */
 }

Desktop Styles

For an individualized wide, or "desktop-style" layout, add the following to the CSS stylesheet.

@media only screen and (min-width: 961px) {
   /* Desktop-only styles go here */
 }

Responsive Web Design Patterns

There are two concepts that web developers must understand regarding how a desktop layout collapses into a mobile layout.

  • A "fluid" layout is one that stretches and shrinks to fill the width of the screen, such as the flexbox layout.
  • A "fixed-width" layout is one that has the same width regardless of the screen dimensions.

Disabling Viewport Zooming

The default behavior for mobile devices is to zoom out of the desktop version far enough so that the entire webpage fits on the screen. To have the mobile layout you created go into effect,add the following element to the <head> of the document. This should be added to every web page you create.

<meta name='viewport' content='width=device-width, initial-scale=1.0, maximum-scale=1.0'/>

Max Width for SVG Images

If an SVG image is too big when you set the width to 100% of its container, then add an inline style to the img tag in the HTML file.

<img class='illustration' src='#' style='max-width: 500px'/>

Max Width for PNG, GIF, and JPG Images

These image files are more sensitive to screen resolution. Keep the inline style the same, but in the CSS file under the width: 100% rule, add display: block;. If there is image distortion, try putting the image into a new <div> container.

Retina Image Optimazation

Since there are two types of device screens, retina and standard, adding the following to the img tag in the HTML file will present the high-resolution image to retina devices and the low-resolution image to standard screens.

<img src='#small-image-file'
      srcset='#small-image-file 1x,
      #big-image-file 2x'
      style='max-width: 500px'/>

Screen Size Optimazation

Using the above optimazation technique means that a retina smartphone will download the high-resolution version, when the standard version is sufficient since the screen is already so small. In this case, you want to optimize the larger image based on the final rendered dimensions rather than the device's screen resolution. Add the following sizes specification to the img tag.

<img src='#small-image-file'
      srcset='#big-image-file 2000w,
      #small-image-file 1000w'
      sizes='(min-width: 960px) 960px, 100vw'/>

The w character is a special unit used only for this image optimazation scenario. The 2000w tells the browser that the big image file is 2000 pixels wide, and the 1000w means that the small image file is 1000 pixels wide.

The sizes specification tells the browser that when the screen is at least 960px wide, the image will also be 960 pixels wide. Otherwise, the 100vw default value tells the browser that the image's width will be 100% of the "viewport width" (aka the screen width).
