CSS Selectors

class

This attribute on the HTML allows you to differentiate between elements of the same type. The same class can be applied to multiple elements in a single HTML document, and one element can be assigned multiple classes (separated by spaces). In this case, the order matters in the stylesheet rather than in the HTML. Classes are signified in the CSS stylesheet with the syntax .name-of-class.

.synopsis em {
   font-style: normal;
 }

Putting the selectors in a particular order allows for greater specification. In this case, only the <em> styles inside of the .synopsis element will be styled normal.

Auto-margin

Styling an element, such as a button or the entire page, with margin: 0 auto; centers the element within the available space.

Pseudo-classes for Links

  • :link - A link the user has never visited.
  • :visited - A link the user has visted before.
  • :hover - A link with the user's mouse over it.
  • :active - A link that is being pressed or clicked.

Use these in the format a:'pseudoclass' to alter features such as text-decoration and color for links.

There are also some pseudo-classes for links that have already been visted.

  • :visited:hover { - A previously visted link that the mouse is hovering over.
  • :visited:active - A previously visited link that is being pressed or clicked

Pseudo-classes for Buttons

To create a button, make an <a href>, including a class attribute to distinguish it as a button.

Formatting is similar to that of links. For buttons, you must format them .button:link in order to override the browser's default a:link style. This can get more specific if you want to differentiate between buttons with classes.

Pseudo-classes for Structure

The :last-of-type pseudo-class selects the final element of a particular type in its parent element. This is useful for things such as creating an extra large margin after the last paragraph in an article.

There is also a :first-of-type pseudo-class respectively.

Note that these select the first or last element in every container element. This can get complicated if you want to style the first paragraph in every container, but there are also paragraphs in a sidebar that you don't want to style. You can avoid this problem by limiting the scope of the pseudo-class.

.page > p:first-of-type {
   color: #7E8184;
   font-style: italic;
 }

This seems to only work for structure pseudo-classes, so narrowing down certain links or buttons doesn't change anything.

id

ID selectors work the same way as class selectors, but you can only have one element with the same ID per page. When you add one of these to the url path of a link as "URL fragment", the browser jumps you down to the part of the webpage that contains the same ID.

ID selectors are styled in CSS like so:

#button-1 {
   color: #7E8184;
 }

Maintaining specifications for a lot of ID selectors is a hassle, so class selectors are used much more often.

CSS Specificity

CSS rules farther down in the stylesheet override rules above them. However, certain CSS selectors that have a higher specificity always win out.

ID selectors have higher specificity than class selectors. Here is a more detailed hierarchy of selectors, from highest to lowest:

  1. #button-2
  2. .button:link
  3. a:link and .synopsis em
  4. .button
  5. a
