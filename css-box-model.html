CSS Box Model

Block Element Rules

  • Block boxes are automatically stacked one after the other according to the order they are laid out on the HTML file.
  • The width of block boxes by default are the width of the parent container.
  • The height of block boxes is based on the content it contains.
  • Inline elements (such as <em>) don't affect vertical spacing, while their width is based on the content it contains.

display

The above rules can be overridden with the display property.

Use display: block; when you're trying to turn <a> elements into buttons or format <img/> elements. It undoes the default inline block display.

Likewise, use display: inline; for changing an element so its features are on the same line.

The CSS Box Model

The CSS box model gives each box four properties:

  • content - The text, image, or other media content in the element.
  • padding - The space between the box's content and its border.
  • border - The line between the box's padding and margin.
  • margin - The space between the box and surrounding boxes.

padding

Only specifying one value creates padding all around the block. For example, saying padding: 50px; adds 50 pixels to each side of the element.

There are shorthand approaches to styling only certain sides of the padding. Stating only two values signifies vertical padding and then horizontal padding. Typing padding: 20px 10px; creates 20 pixels of padding on both the top and bottom of the content and 10 pixels of padding on both the left and right.

Providing four values sets the padding in a clockwise order, starting at the top.

padding: 20px 0 20px 10px; /* Top Right Bottom Left */

border

A border is a line drawn around the content and padding of an element. The syntax of a border is: size style color. Here is an example of creating a border around a block:

border: 1px solid #5d6063;

You can also specify which side of the border.

border-bottom: 1px solid #5d6063;

The border-radius property lets you round the corners of an element's outer border edge.

margin

Margins define the space between a box and its surrounding boxes.

margin-bottom: 50px;

The same rules that apply to padding elements apply to the margin property.

You can differentiate margins from paddings because:

  • The padding of a box has a background, while margins are always transparent.
  • Padding is included in the click area of an element, while margins aren't.
  • Margins collapse vertically, while padding doesn't.

You can center a box by creating auto-margins. Note that this only works on blocks that have an explicit width defined on them. Otherwise, the block will take up the full width of the browser.

margin: 20px auto; /* Vertical Horizontal */

<div> and <span>

Both of these are "container" elements that you can apply box styles to. A <div> is for block-level content while a <span> is for inline content.

width and height

This allows you to explicitly define the width or the height of the content of a block.

box-sizing

Assigning explicit width or height values to a box only affects the box's content. To include the padding and border in the calculation of the box's size, use the box-sizing property to change it from the default content-box to a border-box.

box-sizing: border-box;

Resetting Styles

The browser has a default margin/padding around the page. This differs across different browsers, so it's more efficient to just reset this extra space than to try to accommodate to everything.

* {
   margin: 0;
   padding: 0;
   box-sizing: border-box;
 }
