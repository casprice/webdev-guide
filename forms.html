HTML Forms

HTML forms are used to collect input from your website's visitors. This includes elements such as mailing lists, contact forms, and blog post comments, and the most common types are text input, radio buttons, checkboxes, dropdown menus, text areas, and buttons.

Setting Up a Form

Every HTML form begins with the aptly named <form> element. The most important attributes associated with the <form> are action and method. The action attribute defines the URL that process the form. Whatever information the user inputs before clicking the Submit button is sent to this special URL, usually some sort of Node.js, PHP, or Ruby on Rails. The method attribute can be either post or get. Use post when you're changing data on the server, and get for when you're only getting data.

<form action='/server/' method ='get'>

Text Input Fields

To actually collect user input, you need the <input/> element.

<label for='full-name'>Name</label>

<input id='full-name' name='full-name' type='text'/>

In order to associate the <label> with the specific <input/>, the for attribute in the <label> element must match the id attribute in the <input/> element. The name is like a variable name for the form, and the type identifies the type of form it is. Adding a value prepopulates the form with text.

Styling Text Input Fields

A new type of CSS selector called an attribute selector styles forms. For text input fields, put the following:

input[type='text'] { }

Note that the label can also be individually styled.

To target a specific form, use a descendant selector, such as the div that contains the form.

Email Input Fields

Email inputs work exactly like the type='text' input, except the attributes usually include a placeholder that displays default text when the field is empty. This is a UX technique prompts the user to input their own email.

<label for='email'>Email</label>

<input id='email' name='email' type='email' placeholder='joe@example.com'/>

With the email input field, the user cannot submit the form if it does not contain a valid email. Additionally, for mobile users, the smartphone sees the type='email' attribute and gives the user a special email-specific keyboard with an easily-accessible @ character.

Form Pseudo-classes

To style "right" and "wrong" input values differently to users, use the :invalid and :valid pseudo-classes in the stylesheet. Typically invalid input fields are styled with a red border around the field box. The pseudo-class :focus selects the element that the user is currently filling out.

box-shadow: none; removes the default red glow in Firefox.

Radio Buttons

Changing the type property to radio transforms the <input/> into a radio button. Since these types of forms require multiple selections, you must group them under the same label. Every radio button group you create should:

  • Be wrapped in a <fieldset>, which is labeled with a <legend>.
  • Associate a <label> element with each radio button.
  • Use the same name attribute for each radio button in the group.
  • Use different value attributes for each radio button.

<fieldset class='legacy-form-row'>
  <legend>Type of Talk</legend>
  <input id='talk-type-1'
    name='talk-type'
    type='radio'
    value='main-stage' />
  <label for='talk-type-1' class='radio-label'>Main Stage</label>
  <input id='talk-type-2'
    name='talk-type'
    type='radio'
    value='workshop'
    checked />
  <label for='talk-type-2' class='radio-label'>Workshop</label>
</fieldset>

While the id and the for attributes must match, value is the information sent to the server when the user submits the form. As usual, class groups elements for styling purposes. name groups the same radio button options together.

For styling radio button inputs, it's best to revert back to float styling techniques to adjust the legend name and labels.

Select Menus

The <select> element represents the dropdown menu, and it contains a bunch of <option> elements that represent each item.

<label for='t-shirt'>T-Shirt Size</label>
<select id='t-shirt' name='t-shirt'>
  <option value='xs'>Extra Small</option>
  <option value='s'>Small</option>
  <option value='m'>Medium</option>
  <option value='l'>Large</option>
</select>

Select Menu Font Size

When setting a specific font size in a dropdown menu, stating font-size only really works on Firefox. To fix this for Chrome and Safari, use this vendor-specific prefix beneath the font size rule for the appearance property:

-webkit-appearance: none;

Textareas

The <textarea> element creates a multi=line text field designed to carry large amounts of text from the user. They're suitable for things like biographies, essays, and comments.

<label for='abstract'>Abstract</label>
<textarea id='abstract' name='abstract'></textarea>
<div class='instructions'>Describe in 500 words or less</div>

Styling Textareas

Much of styling textareas is intuitive, but it should be noted that textareas are not input types. Therefore, when singling one out in the stylesheet, simply use the textarea tag.

To prevent the user from being able to resize the <textarea> elements to whatever dimensions they want, use the resize: none; property.

Checkboxes

Checkboxes are sort of like radio buttons, but they allow the user to pick as many options as they want. Since the browser won't have to narrow down which checkboxes are part of the same group, you won't need a <fieldset> wrapper or shared name attributes.

<label class='checkbox-label for='available'>
<input id='available'
  name='available'
  type='checkbox'
  value='is-available'
<span>I'm available this day.</span>
</label>

Submit Buttons

The <button> element represents a button that will submit its containing <form>.
