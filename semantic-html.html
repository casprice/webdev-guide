Semantic HTML

"Semantic HTML" refers to the idea that all of your HTML markup conveys the underlying meaning of your content--whether it's a header, the navigation, etc. They are used as an alternative to simply <div> elements so that search engines, screen readers, and other machines can easier identify the different parts of your website.

<article>

The <article> element represents an independent article in a web page. It should only wrap content that can be plucked out of your page and distributed in a completely different context.

<section>

The <section> element is similar to an <article>, except it doesn't need to make sense outside the context of the document. A <section> can be nested inside of an <article>. These can have their own set of elements, such as headings, independent of the rest of the page.

<nav>

The <nav> element lets you mark up the various navigation sections of your website. This goes for the main site navigation, links to related pages in a sidebar, tables of content--pretty much any group of links. You can include multiple on one page if you have different sets of related links.

<header>

The <header> element denotes introductory content for a section, article, or entire web page. It's best to put a website's name/logo and main navigation here.

<footer>

Footers are like headers, except they belong at the end of an article/website. They can be used for things like copyright noticies, footer navigation, and author bios at the end of blog posts.

<aside>

Asides are put in articles to add in unrelated content, such as an advertisement. They can also contain definitions, stats, or quotations

Dates and Times

The <time> element represents either a time of day or a calendar date. This allows the date an article was posted to appear in the Google search result of the article. Use the following inline HTML tag:

<time datetime='2017-1-3 15:00-0800'>January 3rd</time>

The format of the <time datetime> element goes: Year Month Day Time (24-hour)-Timezone

<address>

The <address> element defines contact information for the author of the article or web page. This is not equivalent to physical addresses.

<address>Please contact <a href='mailto:troymcclure@example.com'>Troy McClure</a> for questions about this article.</address>

Figures and Captions

The <figure> can be used for things such as diagrams, illustrations, or even code snippets. <figcaption> associates a caption with its parent <figure> element.

<figure>
   <img src='image.png'
      alt='Description'/>
   <figcaption>Caption for Image</figcaption>
</figure>
